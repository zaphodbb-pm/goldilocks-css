Comparing to Other CSS Frameworks

Goldilocks-CSS is built using the most up-to-date supported CSS3. Goldilocks-CSS is not backwards compatible with older legacy browsers such as IE. Goldilocks-CSS is positioned to find the balance between comprehensive coverage of the core needs for most designers / developers and sufficient flexibility that includes optional functionality. The designer / developer can choose how much or little of the CSS package they may want to use.

Furthermore, Goldilocks-CSS is built with the end customer in mind. Reduced file size, caching, no javascript required and mobile friendly. Adherence to the latest CSS and HTML standards and best practices ensures compliance with web accessability. Standards compliance and streamlined semantics ensures maintainability and scalability.

Goldilocks-CSS is packaged in four flavours:

  • A core package of layout, and helpers;
  • An extended package of core that also includes common elements;
  • An extended package of core that also includes common elements, icons and components popular with SaaS offerings;
  • An extras package for the extended package and includes elements and components for unique situations;
