Commentary

The magazine component formats a large block of text. This block is typically an article of interest that can be a longer, deeper discussion. To make this more readable, a magazine / newspaper style layout is used to keep the column width from 40 characters (default set at --magazine-width) expanding to fill a column to a maximum of 80 characters before a new column is created.

Modifications to the layout include setting the font family to a serif font. This is usually easier to read. The column width can be set 33% narrower (or about 30ch expanding to 60ch) or 33% wider (about 53ch expanding to 106ch).

Usage

Classes:
.magazine Uses serif font family and is 10% larger than base.
.is-narrow Optional modifier that makes a 33% smaller column width.
.is-wide Optional modifier that makes a 33% wider column width.
.is-highlighted Adds padding and background colour for a paragraph or block.
.mag-block Defines a non-breakable block of content that will not split across columns.
Dependencies:
File: magazine.css
Variables: --bg-highlight, --family-sans-serif, --family-serif, --gutter, --magazine-width, --margin-medium, --padding

Magazine Layout with Serif Fonts, Non-Breakable Block and Narrow Width

Comments

If you have lots of headings and need to keep these headings together with the paragraph text, you will need to wrap the text with a div tag to create a generated box.

The block including the heading can be styled with .mag-block class. This allows the browser to preserve the block and will layout the blocks based on the selected column width. This results in a more natural reading experience.

The block can be highlighted by adding an is-highlighted class to the div tag.
